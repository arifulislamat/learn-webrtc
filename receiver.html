Connection failed — ICE disconnected

Cross-device connections may fail if no usable ICE candidates remain after mDNS filtering. This can happen when the browser blocks all non-mDNS candidates or when the devices are on different networks.

Fixes (pick one):

  1. Use the auto-signaling mode instead — run node server.js and open the Network URL on both devices. Trickle ICE works more reliably across devices.
  2. Disable mDNS anonymization in Brave: go to brave://flags/#enable-webrtc-hide-local-ips-with-mdns → set to Disabled → relaunch Brave.
  3. Ensure both devices are on the same network (same Wi-Fi / LAN). Cross-network connections require a TURN relay server.
  4. Lower Brave Shields for this page (click the lion icon → toggle Shields off).
